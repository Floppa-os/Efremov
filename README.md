# Сайт «Ефремов — новости и события»

Проект отображает актуальную информацию о городе Ефремов: новости, погоду и события.

## Как запустить

1. Скачайте репозиторий.
2. Откройте `index.html` в браузере.

## Структура и назначение файлов проекта «Ефремов — новости и события»


Проект организован в виде набора папок и файлов, каждый из которых выполняет свою роль. Ниже — подробное описание.


### Общая структура папок

```
efremov-city-site/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── main.js
│   └── api.js
├── images/
│   └── logo.png
├── data/
│   └── mock-data.json
└── README.md
```

---

### Подробное описание файлов

#### 1. `index.html`
- **Расположение**: корневая папка проекта.
- **Назначение**: главная страница сайта. Содержит:
  - базовую структуру HTML5‑документа (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`);
  - метаданные (кодировка, адаптивность, заголовок страницы);
  - подключение CSS (`style.css`) и JavaScript‑файлов (`api.js`, `main.js`);
  - семантическую разметку: `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`;
  - контейнеры (`<div id="...">`) для динамического вывода данных (новости, погода, события).
- **Как работает**: при открытии в браузере отображает интерфейс сайта и запускает JS‑скрипты для загрузки данных.

---

#### 2. `css/style.css`
- **Расположение**: папка `css/`.
- **Назначение**: стилизация интерфейса. Отвечает за:
  - сброс стандартных отступов (`margin`, `padding`);
  - базовые шрифты и цвета;
  - оформление шапки (`header`), навигации (`nav`), основного контента (`main`), подвала (`footer`);
  - карточки разделов (`section` с тенью и скруглением);
  - адаптивность (через `box-sizing` и `viewport`).
- **Как подключается**: через тег `<link>` в `<head>` файла `index.html`.

---

#### 3. `js/api.js`
- **Расположение**: папка `js/`.
- **Назначение**: работа с внешними API. Содержит:
  - константы с URL API (новости, погода, события);
  - асинхронную функцию `fetchData(url)` для отправки HTTP‑запросов и обработки ошибок;
  - экспортируемые функции `getNews()`, `getWeather()`, `getEvents()` для вызова из `main.js`.
- **Как используется**: импортируется в `main.js` для получения данных из интернета.

---

#### 4. `js/main.js`
- **Расположение**: папка `js/`.
- **Назначение**: логика отображения данных на странице. Выполняет:
  - вызов функций из `api.js` для получения новостей, погоды и событий;
  - заполнение HTML‑контейнеров (`#news-list`, `#weather-info`, `#events-list`) полученными данными;
  - обработку случаев отсутствия данных (вывод сообщений вроде «Нет новостей»);
  - инициализацию при загрузке страницы через `DOMContentLoaded`.
- **Как подключается**: подключается в конце `<body>` файла `index.html`.


---

#### 5. `images/logo.png`
- **Расположение**: папка `images/`.
- **Назначение**: хранение графических ресурсов. В данном случае — логотип сайта.
- **Как используется**: может быть добавлен в `<header>` файла `index.html` через тег `<img src="images/logo.png" alt="Логотип Ефремова">`.

---

#### 6. `data/mock-data.json`
- **Расположение**: папка `data/`.
- **Назначение**: тестовые данные для разработки. Содержит примеры:
  - новостей (`title`, `date`, `summary`);
  - событий (`name`, `date`, `location`).
- **Как используется**: 
  - на этапе разработки, когда реальные API ещё не подключены;
  - для отладки логики в `main.js` без зависимости от интернета.
- **Формат**: JSON, совместим с ожидаемой структурой данных в JS‑коде.

---

#### 7. `README.md`
- **Расположение**: корневая папка проекта.
- **Назначение**: документация для пользователей и разработчиков. Включает:
  - название и описание проекта;
  - инструкцию по запуску (скачивание, открытие `index.html`);
  - указания по настройке API (например, регистрация на OpenWeatherMap);
  - информацию о структуре проекта.
- **Как используется**: отображается на GitHub как главная страница репозитория.

---

### Как всё взаимодействует

1. **Загрузка страницы**:
   - Браузер открывает `index.html`.
   - Подключаются стили (`style.css`) и скрипты (`api.js`, `main.js`).

2. **Получение данных**:
   - `main.js` вызывает функции из `api.js`.
   - `api.js` отправляет запросы к API (или использует моковые данные из `mock-data.json`).

3. **Отображение данных**:
   - `main.js` вставляет полученные данные в HTML‑элементы через `innerHTML`.
   - Стили из `style.css` оформляют вывод.

4. **Итог**: пользователь видит актуальные новости, погоду и события на странице.

---

### Примечания

- **Для работы с реальными API** замените заглушки в `api.js` на актуальные URL и ключи (например, OpenWeatherMap).
- **Для парсинга сайтов** потребуется серверная часть (Node.js, Python), так как браузерные запросы ограничены CORS.
- **На GitHub** проект будет отображаться как статический сайт (через GitHub Pages), если все файлы загружены в репозиторий.
